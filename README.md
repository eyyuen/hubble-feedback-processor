# Hubble AI Feedback Processor

An AI-powered workflow automation system built for the Hubble Systems Challenge.
Designed to eliminate manual bottlenecks in Hubble's creative agency operations —
starting with the biggest one: unstructured client feedback.

## Demo Video
[Watch Demo Video](https://www.loom.com/share/ca68829531d34ba9a731d813ffae2e30)

## The Problem
Hubble's current workflow has 6 manual bottlenecks:

1. **Unstructured WhatsApp feedback** — PM manually reads every message to decide what to do
2. **Founder bottleneck** — aesthetic and storytelling knowledge locked in one person's head
3. **Sequential blocking** — editors wait for feedback, designers wait for editors
4. **Manual client updates** — PM manually sends status updates every Monday, Wednesday, Friday
5. **Manual follow-ups** — things fall through the cracks when the team is busy
6. **Invoice chaos** — every client billed on a different date, tracked manually

## The Solution — 3-Layer Architecture

Layer 1: Input Capture (planned)
WhatsApp → Twilio API → n8n webhook

Layer 2: AI Brain (prototyped)
Claude AI → categorise, prioritise, assign, draft reply, flag founder review

Layer 3: Automated Outputs (partially built)
Airtable auto-logging → Gmail client updates → Auto follow-ups → Invoice reminders


## What's Built

### AI Feedback Processor (Streamlit app)
Takes raw WhatsApp-style client feedback and returns a fully structured brief:

- **Category** — Revision Request, Approval, New Brief, or Question
- **Priority** — High, Medium, or Low
- **Assigned to** — Editor, Designer, PM, or Founder
- **Issues identified** — extracted from the feedback
- **Action items** — concrete next steps for the team
- **Aesthetic notes** — alignment with Hubble's brand values
- **Founder review flag** — only escalates when genuinely needed
- **Suggested client reply** — professional WhatsApp response, ready to send
- **Airtable auto-logging** — every processed request logged automatically

### What's Planned (Phase 2)
- Twilio WhatsApp webhook → n8n → automatic message capture (no copy-paste)
- Automated MWF client status emails via Gmail
- 24-hour follow-up trigger if no client response
- Per-client invoice reminders via n8n scheduler

## Getting Started

### Prerequisites
- Python 3.9+
- Anthropic API key — [console.anthropic.com](https://console.anthropic.com)
- Airtable account (free) — [airtable.com](https://airtable.com)

### Installation

**1. Clone the repository**
```bash
git clone https://github.com/eyyuen/hubble-feedback-processor.git
cd hubble-feedback-processor
```

**2. Create virtual environment**
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

**3. Install dependencies**
```bash
pip install streamlit anthropic python-dotenv requests
```

**4. Set up environment variables**

Create a `.env` file:
ANTHROPIC_API_KEY=your-anthropic-api-key
AIRTABLE_TOKEN=your-airtable-personal-access-token
AIRTABLE_BASE_ID=your-airtable-base-id
AIRTABLE_TABLE=Feedback

**Airtable setup:**
- Create a base called "Hubble Feedback Tracker"
- Add a table called "Feedback" with these fields:
  - Client Name, Project, Category, Priority, Assigned To (Single line text)
  - Summary, Action Items, Aesthetic Notes, Client Reply, Raw Feedback (Long text)
  - Requires Founder (Checkbox)
  - Turnaround, Received At (Single line text)
- Get your Personal Access Token at [airtable.com/create/tokens](https://airtable.com/create/tokens)
  - Scope: `data.records:write`
  - Select your base

**5. Run the app**
```bash
streamlit run app.py
```

Open `http://localhost:8501`
